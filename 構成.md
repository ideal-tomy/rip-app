# 目的と背景
Zappを利用して、添付のファイルで「複数店舗を繋いだイベント用のテキーラ遠隔注文デモアプリ」を作成しました。
デモが非常に好評だったため、本番運用に向けて本格的な開発とデプロイを行います。

# イベントの運用シーン（重要）
- 各店舗の卓上にQRコードを置き、お客さんが自分のスマホブラウザでアクセスします（Flutter Web）。
- ログイン（メアドやパスワード）は不要です。離脱を防ぐため、初回のみ「居る店舗」と「ニックネーム」を入力して開始します。
- お客さんが「誰に」「何杯」テキーラを撃つか選び、送信ボタンを押します。
- 注文データは、スタッフ用画面（同じアプリ内の管理トグルをONにした状態）に0.1秒でリアルタイム同期されます。
- スタッフは画面を見て対象者にテキーラを提供し、アプリ上で「〇」をタップして「提供済み（✔）」にします。
- 会計システムはアプリに組み込みません。現場のスタッフが直接伝票につけて精算する運用です。

# 技術スタック
- フロントエンド: Flutter Web
- データベース: Firebase Cloud Firestore（リアルタイム同期のため）
- デプロイ先: Firebase Hosting
- 認証: ログイン不要ですが、FirestoreのセキュリティのためにFirebase匿名認証（Anonymous Auth）の導入が必要であれば提案してください。
